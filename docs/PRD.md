## 1. 專案概述

### 1.1 背景與動機

大學生在學期間需要同時處理多門課程的作業、報告、考試準備以及社團活動等各類待辦事項。然而，許多學生缺乏一套有效的方法來追蹤與管理這些任務，導致遺漏截止日期、工作堆積等問題。

本專案旨在開發一套**輕量化的任務管理系統**，讓大學生能夠快速新增、編輯、刪除任務，設定截止日期並標記完成狀態，從而提升個人時間管理與學業效率。

### 1.2 目標用戶

| 用戶角色 | 說明 |
|---------|------|
| **大學生** | 需要管理課堂作業、報告繳交、考試複習等多項學業任務的在學學生 |

### 1.3 核心價值主張

- **簡單易用**：介面直覺，無需學習即可上手，降低使用門檻
- **快速掌握進度**：一眼看出哪些任務已完成、哪些即將到期
- **輕量部署**：基於 Flask + SQLite，無需複雜的伺服器設定即可在本機運行

## 2. 功能需求

### 2.1 功能一覽

| 編號 | 功能名稱 | 說明 |
|------|---------|------|
| F-01 | 新增任務 | 使用者可建立新的任務，輸入任務名稱與相關描述 |
| F-02 | 編輯任務 | 使用者可修改已建立任務的名稱、描述、截止日期等資訊 |
| F-03 | 刪除任務 | 使用者可移除不再需要的任務 |
| F-04 | 標記完成狀態 | 使用者可將任務標記為「已完成」或「未完成」 |
| F-05 | 設定截止日期 | 使用者可為每項任務設定截止日期，方便追蹤時程 |
| F-06 | 任務列表與篩選 | 使用者可瀏覽所有任務，並依完成狀態進行篩選 |
| F-07 | 到期提醒標示 | 系統自動標示已過期或即將到期的任務，以醒目顏色提示 |

### 2.2 使用者故事

#### F-01 新增任務
> 作為一名**大學生**，我希望能夠**快速新增一項任務並填入名稱、描述與截止日期**，以便**將待辦事項記錄下來，避免遺忘**。

**驗收條件：**
- 使用者可在表單中輸入任務名稱（必填）、任務描述（選填）、截止日期（選填）
- 提交後，任務出現在任務列表中
- 名稱為空時顯示錯誤提示，無法送出

#### F-02 編輯任務
> 作為一名**大學生**，我希望能夠**修改已建立的任務內容**，以便**在需求或時程變動時及時更新資訊**。

**驗收條件：**
- 使用者可點擊任務進入編輯頁面
- 可修改任務名稱、描述與截止日期
- 儲存後回到任務列表，內容即時更新

#### F-03 刪除任務
> 作為一名**大學生**，我希望能夠**刪除不再需要的任務**，以便**保持任務列表的整潔與聚焦**。

**驗收條件：**
- 使用者可點擊刪除按鈕移除任務
- 刪除前顯示確認提示，避免誤刪
- 確認後任務從列表中消失

#### F-04 標記完成狀態
> 作為一名**大學生**，我希望能夠**將任務標記為已完成或未完成**，以便**追蹤自己的學習進度與任務達成率**。

**驗收條件：**
- 使用者可點擊核取方塊或按鈕切換任務的完成狀態
- 已完成的任務在視覺上有明顯區別（如刪除線、灰色顯示）
- 狀態切換即時儲存，無需額外確認

#### F-05 設定截止日期
> 作為一名**大學生**，我希望能夠**為任務設定截止日期**，以便**安排優先順序，在期限前完成工作**。

**驗收條件：**
- 新增或編輯任務時可透過日期選擇器設定截止日期
- 任務列表中顯示截止日期
- 截止日期可選擇清除（設定為無期限）

#### F-06 任務列表與篩選
> 作為一名**大學生**，我希望能夠**瀏覽所有任務並依狀態篩選**，以便**快速找到需要處理的待辦事項**。

**驗收條件：**
- 預設顯示所有任務
- 可篩選「全部」、「未完成」、「已完成」三種檢視
- 任務依截止日期排序，最近到期的排在最前面

#### F-07 到期提醒標示
> 作為一名**大學生**，我希望能夠**一眼看出哪些任務即將到期或已逾期**，以便**優先處理緊急的事項**。

**驗收條件：**
- 已過期的任務以紅色標示
- 三天內到期的任務以橘色/黃色標示
- 已完成的任務不受到期提醒影響

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| **後端框架** | Flask (Python) |
| **模板引擎** | Jinja2 |
| **資料庫** | SQLite |
| **前端** | HTML + CSS + JavaScript（原生） |
| **部署方式** | 本機開發環境運行 |

### 3.2 效能考量

- 頁面載入時間應在 **2 秒以內**
- 資料庫查詢應在 **500 毫秒以內** 完成
- 系統應能處理至少 **500 筆任務** 而不影響效能

### 3.3 安全考量

- 所有使用者輸入需進行 **輸入驗證與跳脫處理**（防止 XSS 攻擊）
- 資料庫操作使用 **參數化查詢**（防止 SQL Injection）
- 刪除操作需有 **確認機制**，避免誤操作

### 3.4 可用性考量

- 介面需支援 **繁體中文**
- 設計需具備 **響應式排版**，適合桌面與平板裝置瀏覽
- 操作流程簡潔，主要功能不超過 **3 次點擊** 即可完成

## 4. MVP 範圍

### Must Have（必須實作）

| 功能 | 說明 |
|------|------|
| 新增任務 | 支援輸入名稱、描述、截止日期 |
| 編輯任務 | 可修改所有任務欄位 |
| 刪除任務 | 含刪除確認機制 |
| 標記完成狀態 | 可切換已完成 / 未完成 |
| 任務列表顯示 | 顯示所有任務與基本資訊 |

### Should Have（應該實作）

| 功能 | 說明 |
|------|------|
| 設定截止日期 | 日期選擇器與日期顯示 |
| 到期提醒標示 | 以顏色區分過期、即將到期、正常任務 |
| 任務篩選 | 依完成狀態篩選任務 |

### Nice to Have（如果有時間）

| 功能 | 說明 |
|------|------|
| 任務分類 / 標籤 | 為任務加上分類標籤（如：課業、社團、個人） |
| 排序功能 | 依名稱、日期、狀態等欄位排序 |
| 搜尋功能 | 以關鍵字搜尋任務 |
| 批次操作 | 一次選取多個任務進行刪除或完成 |
